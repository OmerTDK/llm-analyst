# Security Policy

## Scope

This policy covers the `llm-analyst` repository only. There are no live API
credentials, customer PII, or production endpoints in this codebase. The
fixture data (`tests/fixtures/semantic_fixture.duckdb`) is entirely synthetic
(seed=42, generated by the included build tooling).

## Reporting a Vulnerability

Use [GitHub private security advisories](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)
to report any vulnerability:

1. Go to the **Security** tab of this repository.
2. Click **Report a vulnerability**.
3. Describe the issue, reproduction steps, and potential impact.

Do not open a public issue for security vulnerabilities.

**Response commitment:** acknowledgement within 7 days of receipt.
